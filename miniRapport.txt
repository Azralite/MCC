Lors de la création de ce mini compilateur, nous avons été confronté à un problème que nous n'avons pas sur gérer.

Cela concerne le stockage des variables. D'après le squelette de code donné, nous avions accès aux variables via une liste de (string * typ) cependant,
lorsque dans le parser nous voulions ajouter, lire ou modifier une variable il nous était impossible d'y accéder.
Nous avons ensuite eu l'idée de passer par une table de Hashage global, cela fonctionnait mais ne correspondait pas aux attendu de notre language Mini-C.

Nous avons ensuite eu l'idée d'attribuer à chaque table de Hashage une valeur (qu'on qualifierait de "profondeur") avec 0 la table des variables globales et ensuite plus les variables sont locales plus la valeur augmente.
Nous n'avons malheureusement pas eu le temps de l'implémenter.
